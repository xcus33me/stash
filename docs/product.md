# Stash - Product Brief
> Обмен файлами без регистрации, без слежки, с self-hosting возможностью.

---

## Одна фраза 

**Stash** - простой способ поделиться файлами с кем угодно: открыл сайт, создал комнату, загрузил файл, отправил ссылку. Никаких аккаунтов.

---

## Продукт: что входит в v1

### MVP
- загрузка файлов без регистрации
- Уникальные короткие ссылки
- TTL для файлов (1ч / 24ч / 7 дней, etc)
- Комнаты - совместное пространство для файлов, можно использовать под разные контексты
- инвайт ссылки в комнату с коротким TTL
- Экспорт комнаты (для начала экспорт в ZIP)
- Docker-образ для Self-hosting

### In-the-future
- CLI-клиент (`stash upload file.txt`)
- Аналитика для владельца

## Описание бизнес объектов

### Комнаты

Команата - единица хранилища данных. Именно они будут местом хранения данных, определять доступ, и тд.

#### Фичи:
Высокий  приоритет:
- **Описание комнаты + имя файла**: когда получатель открывает ссылку - пусть видит контекст. `Title` + опциональный `Description`.
- **Роли/права на инвайт-ссылке**: 
  - read-only - скачать, но не загрузить
  - upload-only - загрузить, но не видеть чужие файлы (например, сбор работ от студентов, сбор резюме)
  - read-write (defualt)
  - admin (все права)
- **Администрирование без системы аккаунтов**: 
  - owner-токены для владельцев
  - показываются один раз при создании
  - хранится в БД как `owner_token` в таблице `rooms`, hashed
- 
- **Удаление файлов**: владелец должен уметь удалять файлы до истечения TTL файла.
- **Возможность помечать пользователю, кем был отправлен файл**: пользователь сам решает как себя назвать

Средний приоритет:
- `Кто-то загрузил файл` - видим в реальном времени как появляется файл.
- **Lock-режим** - владелец закрывает комнату от новых загрузок.
- **Низкоуровневая настройка комнаты**: 
  - Лимит кол-ва файлов, размера хранилища (1Gb)
  - TTL-файлов
  - Другие
- **Preview файлов в браузере**: Картинки, pdf, текст - без скачивания

Низкий приоритет:
- **Activity Feed** - лента "загружен `file.zip` 3 минуты назад".
